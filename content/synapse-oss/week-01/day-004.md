# Day 4 — Why I built this on 8GB RAM instead of the cloud

## Script (90 seconds)

I built an AI assistant that costs me zero per month to run.

No OpenAI API key required. No Anthropic credits for the default setup. Synapse runs locally — 8GB RAM, Ollama handling LLM inference, everything on the same laptop I use for everything else.

The reasoning was practical, not ideological. Cloud APIs are great until they're not — rate limits, pricing changes, outages at the worst time. And for a personal AI you're trusting with your most private conversations, you probably don't want everything routed through a third-party server anyway.

But local-first has real constraints. The biggest one is RAM. 8GB sounds fine until you're running vector search, a knowledge graph in memory, a 600MB Toxic-BERT model loaded, and Ollama handling inference simultaneously. In the first version, I hit 81% RAM usage. The machine was barely usable for anything else.

I spent three rounds optimizing it. Migrated the knowledge graph from NetworkX to a custom SQLite implementation — cut 155MB immediately. Added lazy loading for the ML models — they unload themselves after 30 seconds of idle time. Moved audio transcription to Groq's cloud Whisper API instead of running it locally.

That brought it down to under 25% on the same hardware.

The cloud fallbacks exist — Gemini, Claude, OpenRouter. But they're optional. The default is local.

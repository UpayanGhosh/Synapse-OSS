# Day 5 — The first version depended on a private binary I couldn't ship

## Script (90 seconds)

I built my app on a dependency I couldn't open source.

Synapse v1's WhatsApp integration ran on OpenClaw — a private binary I didn't write and didn't own. It worked. The bridge between Python and the WhatsApp protocol was solid. But when I decided to make the repo public, I couldn't ship a binary I don't own as part of an open source project.

So I rewrote the entire WhatsApp layer before the repo could go live.

The replacement is Baileys — a Node.js library that implements the WhatsApp Web protocol. It works. But it introduced a new class of problem: two runtimes. Python for the AI logic, Node.js for the WhatsApp bridge, communicating over HTTP.

When the Node process crashes — and it does — Python needs to detect it and recover gracefully. I added exponential backoff reconnection, HTTP heartbeat polling between the two runtimes, and session persistence so the WhatsApp connection doesn't drop completely.

This unplanned rewrite took about 3 weeks.

The lesson: dependencies aren't just libraries. They're commitments. If something your system fundamentally depends on has licensing constraints you don't fully understand, that debt will come due. Mine came due right when I was trying to launch publicly.

```python
@app.get("/api/sessions")
def get_sessions():
    """
    SESS-02: Return session token usage matching openclaw sessions list --json schema.
    Returns last 100 sessions, most recent first.
    Graceful degradation: returns [] if sessions table absent or DB unreadable.
    """
    import sqlite3  # noqa: PLC0415
    from sci_fi_dashboard.db import DB_PATH  # noqa: PLC0415

    try:
        with sqlite3.connect(DB_PATH) as conn:
            conn.row_factory = sqlite3.Row
            rows = conn.execute(
                "SELECT session_id, model, input_tokens, output_tokens, total_tokens, created_at "
                "FROM sessions ORDER BY created_at DESC LIMIT 100"
            ).fetchall()
        return [
            {
                "sessionId": r["session_id"],
                "model": r["model"],
                "inputTokens": r["input_tokens"],
                "outputTokens": r["output_tokens"],
                "totalTokens": r["total_tokens"],
                "contextTokens": 1048576,
                "created_at": r["created_at"],
            }
            for r in rows
        ]
    except Exception:
        return []
```

CRITICAL: Use explicit camelCase key mapping in the return list comprehension. Do NOT return `dict(r)` or `sqlite3.Row` directly — that would return snake_case column names and break state.py consumers.

The route is synchronous (`def`, not `async def`) — it uses blocking sqlite3 which is fine for a lightweight health-adjacent endpoint.

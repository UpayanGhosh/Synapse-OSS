**2. validate_telegram_token(token: str) -> dict**

Rules:
- `r = httpx.get(f"https://api.telegram.org/bot{token}/getMe", timeout=10.0)`
- On `httpx.RequestError`: raise `ValueError(f"Network error reaching Telegram API: {exc}")`
- On `r.status_code == 401`: raise `ValueError("Invalid Telegram token (401 Unauthorized).\nGet a valid token from @BotFather: https://t.me/BotFather")`
- If `r.json().get("ok")` is False: raise `ValueError(f"Telegram API error: {data.get('description', 'unknown')}")`
- Success: return `{"username": result["username"], "id": result["id"]}`

**3. validate_discord_token(token: str) -> dict**

Rules:
- `r = httpx.get("https://discord.com/api/v10/users/@me", headers={"Authorization": f"Bot {token}"}, timeout=10.0)`
- Note: Authorization header MUST use "Bot " prefix (capital B, space after) — this is the Discord API requirement.
- On `httpx.RequestError`: raise `ValueError(f"Network error reaching Discord API: {exc}")`
- On `r.status_code == 401`: raise `ValueError("Invalid Discord bot token (401 Unauthorized).\nGet token from: https://discord.com/developers/applications")`
- On any other non-2xx: raise `ValueError(f"Discord API error {r.status_code}: {r.text[:200]}")`
- Success: return `{"username": data.get("username"), "id": data.get("id")}`

The `setup_discord()` function MUST display the following MESSAGE_CONTENT intent instruction to `console` after a successful token validation call, then call `questionary.press_any_key_to_continue("Press Enter when done...")` (or `input()` as fallback):
```
ACTION REQUIRED — Discord MESSAGE_CONTENT Intent:
  1. Go to: https://discord.com/developers/applications
  2. Select your bot → Bot → Privileged Gateway Intents
  3. Enable: MESSAGE CONTENT INTENT
  Without this, the bot will receive empty messages and disable itself.
```

**4. validate_slack_tokens(bot_token: str, app_token: str) -> dict**

Rules:
- Step 1 prefix check (fail-fast, no network call):
  - `if not bot_token.startswith("xoxb-")`: raise `ValueError(f"Bot token must start with 'xoxb-', got: {bot_token[:12]!r}\nFind it at: Slack App → OAuth & Permissions → Bot User OAuth Token")`
  - `if not app_token.startswith("xapp-")`: raise `ValueError(f"App token must start with 'xapp-', got: {app_token[:12]!r}\nCreate at: Slack App → Basic Information → App-Level Tokens (connections:write scope)")`
- Step 2 `auth.test` for bot_token only (xapp- has NO API validation endpoint):
  - `r = httpx.post("https://slack.com/api/auth.test", headers={"Authorization": f"Bearer {bot_token}"}, timeout=10.0)`
  - On `httpx.RequestError`: raise `ValueError(f"Network error reaching Slack API: {exc}")`
  - If not `data.get("ok")`: map error:
    - `"invalid_auth"` → raise `ValueError("Slack bot token rejected (invalid_auth).\nVerify the xoxb- token in Slack App → OAuth & Permissions.")`
    - other → raise `ValueError(f"Slack auth.test error: {data.get('error', 'unknown')}")`
  - Success: return `{"team": data.get("team"), "user": data.get("user"), "bot_id": data.get("bot_id")}`

**5. run_whatsapp_qr_flow(bridge_dir: str | Path, console: Console | None = None) -> bool**

Implement the synchronous QR flow. All calls are blocking (subprocess.Popen + httpx.get). No asyncio needed:

Constants at module level:
```python
BRIDGE_PORT = 5010
BRIDGE_STARTUP_WAIT = 5.0   # seconds
QR_POLL_INTERVAL = 2.0
QR_TIMEOUT = 30.0
SCAN_TIMEOUT = 120.0
SCAN_POLL_INTERVAL = 3.0
QR_REFRESH_INTERVAL = 30.0  # re-render if QR string changes during polling
```

Logic:
1. Validate Node.js 18+: `shutil.which("node")` → if None, print error + download URL, return False. `subprocess.run(["node", "--version"], capture_output=True, text=True)` → parse major version, if < 18 print error, return False.
2. Check if bridge already running: `httpx.get(f"http://127.0.0.1:{BRIDGE_PORT}/health", timeout=3.0)` — if HTTP 200, reuse running bridge (skip Popen).
3. If not running: `subprocess.Popen(["node", "index.js"], cwd=str(bridge_dir), env={**os.environ, "BRIDGE_PORT": str(BRIDGE_PORT), "PYTHON_WEBHOOK_URL": "http://127.0.0.1:8000/channels/whatsapp/webhook"}, stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL)`
4. Wait BRIDGE_STARTUP_WAIT seconds.
5. Poll `GET /qr` with deadline = now + QR_TIMEOUT. If no QR within timeout, print error, return False (in finally).
6. Render QR: `qr = qrcode.QRCode(error_correction=qrcode.constants.ERROR_CORRECT_L); qr.add_data(qr_string); qr.make(fit=True); f = io.StringIO(); qr.print_ascii(out=f, invert=True); f.seek(0); ascii_art = f.read()`
7. Print QR with rich Panel or plain print. Show: "SCAN WITH WHATSAPP → Linked Devices → Link a Device" and "Waiting for scan (120 second timeout)..."
8. Poll `GET /health` with deadline = now + SCAN_TIMEOUT every SCAN_POLL_INTERVAL. If `connectionState == "connected"` → print success, return True. If `connectionState == "logged_out"` → print error, return False.
9. On timeout: print timeout message, return False.
10. ALWAYS: `proc.terminate(); proc.wait(timeout=5)` in `finally` block (only if bridge was started by wizard, not if it was pre-existing).

Windows QR fix: before printing, if `sys.platform == "win32"`: `import sys as _sys; _sys.stdout.reconfigure(encoding='utf-8')` to prevent garbled box-drawing characters.

**6. setup_telegram(non_interactive: bool = False) -> dict | None**

If non_interactive: read token from `os.environ.get("SYNAPSE_TELEGRAM_TOKEN")`. If absent: return None.
If interactive: `questionary.select("Telegram:", choices=["Configure", "Skip"]).ask()` → None if Skip.
Prompt: `questionary.password("Enter Telegram bot token:").ask()` (or env var in NI mode).
Validate: `validate_telegram_token(token)` in `console.status(...)` spinner.
If ValueError: print error, return None.
Return: `{"token": token}` on success.

**7. setup_discord(non_interactive: bool = False) -> dict | None**

Same skip-gate pattern.
Collect token. Validate. Show MESSAGE_CONTENT intent instruction (even in NI mode — print it, no pause).
Collect optional `allowed_channel_ids`: `questionary.text("Allowed channel IDs (comma-separated, or leave blank for all):").ask()` → parse to `list[int]`.
Return: `{"token": token, "allowed_channel_ids": channel_ids}` on success.

**8. setup_slack(non_interactive: bool = False) -> dict | None**

Same skip-gate pattern.
Collect bot_token (xoxb-) and app_token (xapp-).
Show instruction about `connections:write` scope for app_token.
Validate with validate_slack_tokens().
Return: `{"bot_token": bot_token, "app_token": app_token}` on success.

If interactive: skip gate → run_whatsapp_qr_flow(bridge_dir). If returns True: return `{"enabled": True, "bridge_port": BRIDGE_PORT}`. If False: return None.

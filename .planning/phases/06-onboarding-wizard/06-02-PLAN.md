Continue workspace/cli/provider_steps.py by adding:

**1. validate_provider(provider: str, api_key: str) -> ValidationResult**

Rules:
- Call `litellm.acompletion()` with `model=VALIDATION_MODELS[provider]`, `messages=[{"role":"user","content":"Hi"}]`, `max_tokens=1`, `timeout=5.0`, `num_retries=0`
- Before the call: save existing env var value, set env var to api_key. After: restore (even on exception). Pattern:
  ```python
  env_var = _KEY_MAP[provider]
  old = os.environ.get(env_var)
  os.environ[env_var] = api_key
  try:
      ...
  finally:
      if old is None: os.environ.pop(env_var, None)
      else: os.environ[env_var] = old
  ```
- Exception mapping:
  - `AuthenticationError` → ValidationResult(ok=False, error="invalid_key", detail=str(e))
  - `RateLimitError` → ValidationResult(ok=True, error="quota_exceeded", detail=str(e))  # KEY IS VALID
  - `Timeout` → ValidationResult(ok=False, error="timeout", detail=str(e))
  - `APIConnectionError` → ValidationResult(ok=False, error="network_error", detail=str(e))
  - `BadRequestError` → ValidationResult(ok=False, error="bad_request", detail=str(e))
  - Any other Exception → ValidationResult(ok=False, error="unknown", detail=str(e))
- CRITICAL: always set max_tokens=1. Anthropic rejects requests without max_tokens.
- The function is sync; it calls `asyncio.run(_validate_async(...))` internally.
- Do NOT raise exceptions — always return ValidationResult.

**2. validate_ollama(api_base: str = "http://localhost:11434") -> ValidationResult**

Rules:
- Use `httpx.get(f"{api_base}/api/version", timeout=5.0)` (sync, no asyncio needed)
- HTTP 200 → ValidationResult(ok=True, detail=f"Ollama {resp.json().get('version','unknown')} running")
- Any non-200 → ValidationResult(ok=False, error="http_error", detail=f"HTTP {resp.status_code}")
- `httpx.ConnectError` → ValidationResult(ok=False, error="not_running", detail=f"Ollama not reachable at {api_base}")
- `httpx.TimeoutException` → ValidationResult(ok=False, error="timeout", detail="5s timeout")
- Do NOT call litellm for Ollama. No API key needed.

**3. github_copilot_device_flow(console) -> str | None**

Rules:
- This is an async function called via `asyncio.run()` from the wizard.
- Step 1: POST to `https://github.com/login/device/code` with json=`{"client_id": GITHUB_CLIENT_ID, "scope": "read:user"}` and header `Accept: application/json`. Extract: device_code, user_code, verification_uri, interval.
- GITHUB_CLIENT_ID = `"Iv1.b507a08c87ecfe98"` (public GitHub Copilot client_id per litellm community sources — verify in llm_router.py at implementation time, fall back to this value if not found)
- Step 2: Print to console: "Visit {verification_uri}" and "Enter code: {user_code}". Call `webbrowser.open(verification_uri)`.
- Step 3: Poll `https://github.com/login/oauth/access_token` every `interval` seconds (max 5 minutes / 300 seconds):
  - `"access_token"` key present → return the token
  - `error == "authorization_pending"` → continue
  - `error == "slow_down"` → increment interval by 5, continue
  - Any other `error` → break (expired / denied)
- After receiving token: write to `Path(os.environ.get("GITHUB_COPILOT_TOKEN_DIR", Path.home() / ".config/litellm/github_copilot")) / "api-key.json"` with structure:
  ```json
  {"token": "<access_token>", "expires_at": time.time() + 28800, "endpoints": {"api": "https://api.githubcopilot.com"}}
  ```
- Return token string on success, None on timeout/denial.
- Use `httpx.AsyncClient` (async, inside async function).
- Timeout/denial: console.print error message, return None.

All three functions must be defined in workspace/cli/provider_steps.py. The file should import: `os`, `asyncio`, `time`, `json`, `webbrowser`, `httpx`, `litellm`, `from litellm import AuthenticationError, RateLimitError, APIConnectionError, Timeout, BadRequestError`, `from pathlib import Path`.

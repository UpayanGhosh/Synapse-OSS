<task type="auto">
  <name>Task 2: Create baileys-bridge/index.js — Express + Baileys microservice</name>
  <files>baileys-bridge/index.js</files>
  <action>
Create baileys-bridge/index.js. This is a CommonJS file (~150-160 lines). Follow the exact patterns from the research document.

**Structure:**
1. Require all dependencies (express, NodeCache, baileys, writeFileAtomic, qrcode, pino)
2. Read env vars: BRIDGE_PORT (default 5010), PYTHON_WEBHOOK_URL (default 'http://127.0.0.1:8000/channels/whatsapp/webhook')
3. Initialize NodeCache for group metadata (stdTTL: 300, useClones: false)
4. Module state: `let sock = null; let qrData = null; let connectionState = 'disconnected';`
5. `async function atomicSaveCreds(creds)` — writes creds JSON with writeFileAtomic, then copies auth_state/ to auth_state.bak/ via fs.cpSync
6. `async function forwardToFastAPI(payload)` — uses built-in global fetch() (Node 18+, not node-fetch npm) to POST to PYTHON_WEBHOOK_URL with AbortSignal.timeout(5000)
7. `function extractPayload(msg)` — returns {channel_id: 'whatsapp', user_id, chat_id, text, message_id, is_group, timestamp, raw}
8. `async function startSocket()`:
   - Call `useMultiFileAuthState('./auth_state')` to get {state, saveCreds}
   - Create atomic saveCreds wrapper that calls writeFileAtomic for each creds file then backups auth_state/
   - Call makeWASocket with: auth: state, logger: pino({level: 'silent'}), printQRInTerminal: false, cachedGroupMetadata: async (jid) => groupCache.get(jid), markOnlineOnConnect: false
   - Register sock.ev.on('creds.update', atomicSaveCredsWrapper)
   - Register sock.ev.on('groups.update', ...) — calls sock.groupMetadata(event.id) and sets in groupCache
   - Register sock.ev.on('group-participants.update', ...) — same pattern
   - Register sock.ev.on('connection.update', ...) — handles QR display, 'open'/'close' states; CRITICAL: on close check DisconnectReason.loggedOut (401) and DisconnectReason.forbidden (403) — set connectionState='logged_out', log CRITICAL, do NOT reconnect; for other codes (restartRequired 515, connectionClosed 428, connectionLost/timedOut 408) call startSocket() to restart socket only (not Express server)
   - Register sock.ev.on('messages.upsert', ...) — skip type !== 'notify', skip msg.key.fromMe, call extractPayload(msg) and forwardToFastAPI(payload)
9. Express app setup: app.use(express.json()), register routes, app.listen(PORT)
10. Routes:
    - POST /send: check sock && connectionState==='connected', add 1000 + Math.random()*2000 ms delay (anti-spam jitter), call sock.sendMessage(jid, {text}), return {ok: true} or 503/500
    - POST /typing: call sock.sendPresenceUpdate('composing', jid), return {ok: true}
    - POST /seen: call sock.readMessages([{remoteJid: jid, id: messageId, fromMe, participant}]), return {ok: true}
    - GET /health: return {status: connectionState==='connected' ? 'ok' : 'degraded', connectionState, pid: process.pid}
    - GET /qr: if no qrData return 404 {error: 'No QR available'}; else return {qr: qrData}
11. Call startSocket() at bottom of file

**Critical correctness requirements:**
- Use require() not import/export — CJS, not ESM
- NEVER use 'node-fetch' npm — Node 18+ built-in global fetch() is available since WA-08 validates 18+
- The DisconnectReason import: `const { default: makeWASocket, useMultiFileAuthState, DisconnectReason } = require('@whiskeysockets/baileys');`
- atomicSaveCreds must use writeFileAtomic — not fs.writeFile
- Set printQRInTerminal: false — we serve QR via GET /qr ourselves; qrcode-terminal is still used to print to terminal for convenience only (not for the API)
- qrData stores the raw QR string (from `qr` in connection.update) — returned as-is in GET /qr
